## Data Flow

### Catch Lifecycle
```
Fisher submits catch
  вҶ’ API validates input (Zod)
  вҶ’ Stored in PostgreSQL (status: pending)
  вҶ’ Queued in BullMQ (catch-validation)
  вҶ’ FraudDetector checks: duplicate window, weight, GPS, vessel status
  вҶ’ If clean: submitted to CatchVerification Soroban contract
  вҶ’ Verifier reviews on Regulator dashboard
  вҶ’ On approval: BlueCreditMinting contract mints credits
  вҶ’ Settlement contract queues payout
  вҶ’ Fisher receives XLM via Stellar anchor
```

### Telemetry Flow
```
GPS device / SMS вҶ’ POST /api/telemetry
  вҶ’ BullMQ telemetry queue (concurrency: 50)
  вҶ’ Stored in PostgreSQL telemetry table
  вҶ’ FraudDetector checks implied speed vs last known position
  вҶ’ Anomaly вҶ’ compliance_score decremented on vessel
```

### SMS Offline Flow
```
Fisher sends SMS вҶ’ Africa's Talking webhook вҶ’ POST /api/sms/incoming
  вҶ’ State machine: INIT вҶ’ VESSEL_ID вҶ’ SPECIES вҶ’ WEIGHT вҶ’ CONFIRM
  вҶ’ On YES: catch inserted + queued for validation
  вҶ’ Confirmation SMS sent back to fisher
```

## Module Responsibilities

| Module | Responsibility |
|--------|---------------|
| `contracts/vessel-registry` | Vessel identity, compliance scores, role management |
| `contracts/catch-verification` | Catch submission, duplicate guard, verifier approval |
| `contracts/blue-credit-minting` | Sustainability scoring, soulbound credit issuance |
| `contracts/settlement` | Payout queuing, batch execution via Stellar token |
| `contracts/esg-index` | Periodic ESG snapshots with Merkle proof |
| `backend/api-gateway` | REST API, queue workers, fraud detection, SMS gateway |
| `frontend/web` | Four role-based dashboards |
| `sdk` | TypeScript client library for all API surfaces |

## Role-Based Access

| Role | Permissions |
|------|-------------|
| `fisher` | Submit catches, view own data, log telemetry |
| `vessel_owner` | Register vessels, submit catches |
| `verifier` | Approve/reject catches, mint credits, queue payouts |
| `esg_auditor` | Publish ESG snapshots |
| `admin` | All permissions |

## Security Design

- **JWT authentication** вҖ” Stellar wallet challenge/response (Ed25519 signature)
- **Rate limiting** вҖ” 200 req/min per IP on API gateway
- **Duplicate guard** вҖ” On-chain (Soroban temporary storage) + off-chain (DB window query)
- **Fraud detection** вҖ” GPS speed analysis, weight anomaly, vessel status check
- **Role enforcement** вҖ” Both on-chain (contract panics) and off-chain (middleware)
- **Audit trail** вҖ” All contract calls emit events; all API calls logged via pino
